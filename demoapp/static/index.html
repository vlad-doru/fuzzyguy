Explanations

The service was developed in Go because the language facilitates the development of highly concurrent systems.

We use locality-sensitive hash functions in order to find the nearest neighbours in the metric space determined by the Levenshtein distance. One of these functions counts the parity of the numbers of characters which are mapped to a specific location between 0 and 31.

With the two hash values computed we can then find a lower bound for the Levenshtein distance between the two strings. This lower bound is given by the number of different set bits in the histograms to which we add the length difference of the strings and then we divide everything by 2.

Using the hash function previously described, the service uses bit-wise operations to increase performance.
